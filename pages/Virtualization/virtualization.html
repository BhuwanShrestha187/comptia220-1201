üìù Practice Questions

Question 1: What is the difference between Type 1 and Type 2 hypervisors?

Answer: Type 1 hypervisors (bare-metal) run directly on hardware and are used in enterprise data centers. Type 2 hypervisors (hosted) run as software on a host operating system and are used for development and testing.

Question 2: In the shared responsibility model, who manages the operating system in an IaaS deployment?

Answer: The customer manages the operating system in IaaS. The provider manages the underlying virtualization, servers, storage, and networking infrastructure.

Question 3: What cloud characteristic allows resources to automatically scale up or down based on demand?

Answer: Elasticity. This enables cost-efficient handling of variable workloads by provisioning and deprovisioning resources as needed.

üéØ Key Takeaways

  • Hypervisors: Type 1 (ESXi, Hyper-V) for production; Type 2 (VirtualBox, VMware Workstation) for testing
  • Cloud models: Public (shared), Private (dedicated), Hybrid (combination), Community (shared orgs)
  • Service models: IaaS (most control), PaaS (developers), SaaS (end users)
  • Cloud benefits: Elasticity, scalability, pay-as-you-go, managed services
  • Consider egress costs and data residency when designing cloud solutions
