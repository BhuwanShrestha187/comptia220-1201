CompTIA A+ 220-1101 - Drive Configurations

What is RAID?

RAID = Redundant Array of Independent (or Inexpensive) Disks

Combines multiple physical drives into one logical unit for:

  • Performance: Faster read/write speeds
  • Redundancy: Data protection against drive failure
  • Both: Some RAID levels provide speed AND protection

RAID Levels You MUST Know

RAID 0 - Striping

Minimum Drives:
2
Redundancy:
NONE
Usable Capacity:
100% (all drives)
Performance:
Excellent (fastest)

How it works:

  • Data split (striped) across all drives
  • Each drive holds part of the data
  • All drives work together simultaneously

Benefits:

  • Maximum performance - reads and writes are faster
  • Full capacity of all drives available
  • Simple to set up

Drawbacks:

  • NO FAULT TOLERANCE! If ANY drive fails, ALL data is lost
  • Not for critical data

Best Use: Gaming PCs, video editing (where speed matters, not data safety)

Example: Two 1TB drives in RAID 0 = 2TB usable space, fast performance, but if one drive fails, everything is lost.

RAID 1 - Mirroring

Minimum Drives:
2
Redundancy:
YES (full mirror)
Usable Capacity:
50% (size of one drive)
Performance:
Good reads, normal writes

How it works:

  • Identical copy (mirror) of data on each drive
  • Both drives contain exactly the same data
  • If one fails, the other has complete data

Benefits:

  • Complete redundancy - one drive can fail
  • Simple to understand and implement
  • Fast read performance (can read from both drives)

Drawbacks:

  • 50% capacity loss - you only get space of one drive
  • More expensive per usable TB

Best Use: Boot drives, critical data that must be protected

Example: Two 1TB drives in RAID 1 = 1TB usable space. If one drive fails, data is safe on the other.

RAID 5 - Striping with Parity

Minimum Drives:
3
Redundancy:
YES (1 drive failure)
Usable Capacity:
(N-1) drives worth
Performance:
Good reads, slower writes

How it works:

  • Data striped across all drives (like RAID 0)
  • Parity information distributed across all drives
  • Parity allows data reconstruction if one drive fails

Benefits:

  • Good balance of performance, redundancy, and capacity
  • Can survive one drive failure
  • Better capacity efficiency than RAID 1

Drawbacks:

  • Lose capacity of one drive to parity
  • Write performance slower (must calculate parity)
  • If second drive fails during rebuild, all data lost
  • Rebuild time can be long

Best Use: File servers, NAS devices, business data storage

Example: Three 1TB drives in RAID 5 = 2TB usable space (1TB for parity). One drive can fail with no data loss.

RAID 10 (1+0) - Mirrored Stripes

Minimum Drives:
4
Redundancy:
YES (multiple drives)
Usable Capacity:
50% (like RAID 1)
Performance:
Excellent (best of RAID 0 & 1)

How it works:

  • Creates mirrored pairs (RAID 1)
  • Then stripes across the pairs (RAID 0)
  • Combines benefits of RAID 0 and RAID 1

Benefits:

  • Excellent performance (striping) with redundancy (mirroring)
  • Can survive multiple drive failures (if not in same mirror pair)
  • Faster rebuild than RAID 5

Drawbacks:

  • Expensive - only get 50% capacity
  • Requires minimum 4 drives

Best Use: Database servers, high-performance applications needing redundancy

Example: Four 1TB drives in RAID 10 = 2TB usable. Two drives mirrored, then striped with another mirrored pair.

Quick RAID Comparison

RAID Level Min Drives Capacity Fault Tolerance Performance Best For
RAID 0 2 100% None Fastest Speed, non-critical data
RAID 1 2 50% 1 drive Good reads Critical data, boot drives
RAID 5 3 N-1 drives 1 drive Good reads, slower writes File servers, NAS
RAID 10 4 50% Multiple drives Excellent Databases, high performance

Other RAID Levels (Less Common)

RAID 6: Like RAID 5 but with dual parity - can survive TWO drive failures. Requires minimum 4 drives. Uses capacity of 2 drives for parity.

JBOD (Just a Bunch Of Disks): Not really RAID. Combines drives into one volume but no striping or redundancy. Each drive fills sequentially.

Important RAID Concepts

Hot Spare

  • Extra drive that sits idle in the RAID array
  • Automatically activates when a drive fails
  • Rebuild starts immediately without technician intervention
  • Common in enterprise environments

Rebuild Process

  • When a drive fails in RAID 1, 5, or 10, replace the failed drive
  • RAID controller rebuilds data onto new drive
  • Array remains in degraded state during rebuild
  • System still accessible but at risk - second failure is catastrophic
  • Can take hours to days depending on drive size
‚ö†Ô∏è Critical: RAID is NOT a Backup!
  • RAID protects against hardware failure only
  • Does NOT protect against: deletion, corruption, virus, fire, theft
  • Always maintain separate backups of critical data
  • Follow 3-2-1 backup rule: 3 copies, 2 different media, 1 offsite
