üîÑ TCP vs UDP

  • TCP (Transmission Control Protocol):
  • ‚Ä¢ Connection-oriented with three-way handshake
  • ‚Ä¢ Error checking and delivery confirmation
  • ‚Ä¢ Guaranteed delivery, ordered packets
  • ‚Ä¢ Higher overhead, slower than UDP
  • ‚Ä¢ Used for: HTTP, HTTPS, FTP, SMTP, RDP
  • UDP (User Datagram Protocol):
  • ‚Ä¢ Connectionless, no handshake
  • ‚Ä¢ No error checking or delivery guarantee
  • ‚Ä¢ Lower overhead, faster transmission
  • ‚Ä¢ Used for: DNS, DHCP, SNMP, VoIP, streaming

üì° DNS Record Types

  • A Record: Maps hostname to IPv4 address
  • AAAA Record: Maps hostname to IPv6 address
  • CNAME: Canonical name (alias for another name)
  • MX Record: Mail exchanger (email server)
  • TXT Record: Text information for verification
  • NS Record: Name server delegation
  • SOA Record: Start of Authority zone info

2.2 Wireless Networking Technologies

Wireless networking uses radio frequencies to transmit data without physical cables. Understanding wireless standards and configurations is essential.

üì∂ Wi-Fi Standards (802.11)

  • 802.11a: 5GHz, up to 54Mbps (legacy)
  • 802.11b: 2.4GHz, up to 11Mbps (legacy)
  • 802.11g: 2.4GHz, up to 54Mbps (legacy)
  • 802.11n (Wi-Fi 4): 2.4/5GHz, up to 600Mbps
  • 802.11ac (Wi-Fi 5): 5GHz, up to 6.9Gbps
  • 802.11ax (Wi-Fi 6/6E): 2.4/5/6GHz, up to 9.6Gbps

üì° Frequency Bands

  • 2.4GHz: Longer range, better penetration, crowded
  • 5GHz: Shorter range, less interference, faster
  • 6GHz (Wi-Fi 6E): Newest, least crowded, fastest
  • Channels: Non-overlapping channels needed
  • DFS: Dynamic Frequency Selection (radar avoidance)

üîê Wireless Security

  • WEP: Old, insecure (avoid)
  • WPA: Temporary fix, TKIP encryption
  • WPA2: AES encryption, common standard
  • WPA3: Newest, improved security
  • Enterprise: 802.1X with RADIUS server

üì± Bluetooth

  • Range: Class 2 (~10m), Class 1 (100m)
  • Frequency: 2.4GHz ISM band
  • Versions: 5.0, 5.1, 5.2, 5.3, 5.4
  • Uses: Audio, peripherals, IoT devices
  • BLE: Bluetooth Low Energy for sensors

üè∑Ô∏è RFID

  • Active RFID: Battery-powered, longer range
  • Passive RFID: No battery, short range
  • Uses: Asset tracking, access control
  • Frequencies: Low (LF), High (HF), Ultra (UHF)

üì≥ NFC

  • Range: ~4cm (1.5 inches)
  • Frequency: 13.56MHz
  • Uses: Payments, pairing, data transfer
  • Modes: Reader, P2P, Card emulation

2.4 Network Hardware Devices

Various hardware devices form the infrastructure of computer networks. Understanding their functions is crucial for network setup and troubleshooting.

üîÑ Network Devices

  • Router: Connects multiple networks, routes traffic
  • Switch: Connects devices on same network (Layer 2)
  • Managed Switch: Configurable, supports VLANs
  • Unmanaged Switch: Plug-and-play, no configuration
  • Access Point: Provides wireless network access
  • Firewall: Security device, filters traffic

üîå Connectivity Hardware

  • NIC (Network Interface Card): Connects device to network
  • MAC Address: Unique hardware identifier (48-bit)
  • Patch Panel: Central termination point for cables
  • PoE Injector: Adds power to Ethernet cable
  • PoE Switch: Switch with built-in PoE capability
  • ONT: Optical Network Terminal (fiber)

üì° Modems and Connectivity

  • Cable Modem: Uses coaxial cable infrastructure
  • DSL Modem: Uses telephone lines (ADSL/VDSL)
  • Fiber ONT: Converts fiber to Ethernet
  • Satellite Modem: For remote areas
  • Cellular Gateway: Provides cellular data connectivity

üåê Network Types

  • LAN: Local Area Network (single building)
  • WAN: Wide Area Network (geographic)
  • WLAN: Wireless LAN
  • PAN: Personal Area Network (Bluetooth)
  • MAN: Metropolitan Area Network (city)
  • SAN: Storage Area Network

2.5 SOHO Network Configuration

Small Office/Home Office networks require proper configuration of IP addressing and internet connectivity settings.

üåê IP Addressing

  • IPv4: 32-bit address (e.g., 192.168.1.100)
  • IPv6: 128-bit address (e.g., 2001:db8::1)
  • Private Addresses:
  • ‚Ä¢ 10.0.0.0 - 10.255.255.255 (/8)
  • ‚Ä¢ 172.16.0.0 - 172.31.255.255 (/12)
  • ‚Ä¢ 192.168.0.0 - 192.168.255.255 (/16)
  • APIPA: 169.254.0.0/16 (auto-assigned when DHCP fails)

üìä IP Configuration Methods

  • Static IP: Manually configured address
  • Dynamic IP: Assigned by DHCP server
  • DHCP Lease: Temporary assignment with expiration
  • Reservation: Fixed IP for specific MAC address
  • Scope: Range of available IP addresses
  • Exclusion: Addresses not available in scope

üîó Network Configuration

  • Subnet Mask: Defines network vs host portion
  • Default Gateway: Router address for external traffic
  • DNS Server: Resolves domain names
  • VPN: Virtual Private Network for secure remote access
  • VLAN: Virtual LAN for network segmentation

üì∂ Internet Connection Types

  • Cable: Coaxial cable, high speed, shared
  • DSL: Telephone line, asymmetric speeds
  • Fiber: Optical cable, fastest speeds
  • Satellite: For remote areas, latency issues
  • Cellular: 4G/5G mobile broadband
  • WISP: Wireless Internet Service Provider

2.6 Networking Tools

Various tools are used for cable installation, testing, and network troubleshooting.

üîß Cable Tools

  • Crimper: Attaches RJ45/RJ11 connectors to cable
  • Cable Stripper: Removes cable jacket without damaging wires
  • Punchdown Tool: Terminates cable to keystone jack/patch panel
  • Cable Tester: Verifies connectivity and wiring order
  • Toner Probe: Traces cables through walls
  • Loopback Plug: Tests NIC by returning signals

üì° Wireless Tools

  • Wi-Fi Analyzer: Shows networks, signal strength, channels
  • Packet Analyzer (Wireshark): Captures/analyzes traffic
  • Speed Test: Measures network throughput
  • IP Scanner: Discovers devices on network

üíª Command Line Tools

  • ipconfig/ifconfig: View IP configuration
  • ping: Test connectivity to host
  • tracert/traceroute: Trace route to destination
  • nslookup/dig: DNS lookup tools
  • netstat: View network connections

üìù Practice Questions

Question 1: Which port does HTTPS use by default?

Answer: Port 443 (TCP). This is the standard port for secure web traffic using SSL/TLS encryption.

Question 2: What is the difference between a managed and unmanaged switch?

Answer: A managed switch can be configured (VLANs, QoS, port security) while an unmanaged switch is plug-and-play with no configuration options.

Question 3: A user gets an IP address in the 169.254.0.0/16 range. What does this indicate?

Answer: APIPA (Automatic Private IP Addressing) - the device could not reach a DHCP server, so it assigned itself a link-local address.

üéØ Key Takeaways

  • Common ports: 20-21 (FTP), 22 (SSH), 25 (SMTP), 53 (DNS), 80 (HTTP), 443 (HTTPS), 3389 (RDP)
  • TCP is reliable with error checking; UDP is faster but unreliable
  • Wi-Fi 6 (802.11ax) is the latest standard, operating on 2.4, 5, and 6GHz
  • SOHO networks typically use DHCP with private IP ranges (192.168.x.x)
  • Essential tools include crimpers, cable testers, and punchdown tools
